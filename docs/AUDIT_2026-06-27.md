**Findings: 6 CRITICAL, 5 HIGH, 9 MEDIUM, 4 LOW**

## 🔴 CRITICAL

### C1 — DMA buffer overflow via PRD byte_count underflow (ahci.rs:549)
`(byte_count - 1) | 1` underflows to `0xFFFFFFFF` when `byte_count == 0`, causing 4 GiB DMA write from a small bounce buffer. Ring-0 DMA corruption.
**Fix:** Add `byte_count > 0` guard before writing PRD.

### C2 — NIC RX descriptor length not clamped (e1000.rs:354-358)
Hardware-provided `length` field from RX descriptor is used unchecked to create a slice. An attacker-controlled NIC can set length up to 65535, causing OOB read past the 2048-byte DMA buffer — kernel memory leak + potential panic.
**Fix:** Clamp `len` to `min(len, RX_BUFFER_SIZE)`.

### C3 — SYS_KILL has no permission check (syscall.rs:733-736)
Any user task can kill any process, including PID 1 (init), by specifying the target PID. No capability check, no UID/ownership check.
**Fix:** Add task ownership or capability check before kill.

### C4 — IPC `send()` disables interrupts unconditionally, `.expect()` on missing endpoint panics with interrupts off (ipc.rs:137-148)
If the endpoint doesn't exist, `.expect()` panics with interrupts disabled. The panic handler can't preempt because IF=0. The kernel hangs forever (deadlock, never reaches hlt loop).
**Fix:** Use proper error handling, not `.expect()`, or use RAII interrupt guard.

### C5 — IPC `call()` busy-spins with interrupts disabled while waiting for reply (ipc.rs:205-219)
`call()` enters a loop that polls the reply slot with interrupts disabled. On single-core, the timer ISR can't fire, so no other task runs — the system hangs forever on an unresponded call.
**Fix:** Re-enable interrupts between polling iterations; use a blocking wait mechanism.

### C6 — Signals can't wake tasks blocked on IPC receive (task.rs:975, ipc.rs:151-183)
`do_kill` only wakes tasks blocked on `wait()`. A task blocked in `ipc::receive()` with `State::Blocked` but `wait_blocked=false` will never wake for a signal. The kill signal is lost.
**Fix:** Add IPC wake path to `do_kill`, or implement async signal delivery.

## 🟠 HIGH

### H1 — No capability enforcement anywhere except `SYS_CAP_CHECK` (syscall.rs, capability.rs)
The capability system exists but only one syscall uses it. All other operations (kill, file ops, IPC, mmap) bypass capabilities entirely. Ambient authority by default.
**Fix:** Wire capability checks into SYS_KILL, SYS_MMAP, SYS_OPEN, IPC.

### H2 — SYS_MUNMAP no ownership validation (syscall.rs:272-304)
`munmap` unmaps any user-space address without checking if the task actually owns those pages. Task can unmap pages it shouldn't — self-DoS.
**Fix:** Track per-task mapped regions and verify ownership.

### H3 — `do_wait` writes exit status to unvalidated user pointer (task.rs:559-562)
`do_wait` dereferences `status_ptr` (user-provided pointer) without validation. A user task can make the kernel write an `i32` to any kernel address by passing it as the status pointer.
**Fix:** Validate `status_ptr` with `is_user_pointer()` before write.

### H4 — Waiter duplicates accumulate in IPC receive loop (ipc.rs:167-183)
On spurious wake-up, `receive()` loops back, checks queue, and re-registers as a waiter. The old waiter entry remains, causing duplicates and unbounded `VecDeque` growth.
**Fix:** Remove stale waiter entry before re-looping.

### H5 — TX bounds check uses wrong constant name (e1000.rs:301)
`data.len() > RX_BUFFER_SIZE` should be `TX_BUFFER_SIZE`. Currently safe because both = 2048, but a maintenance hazard.
**Fix:** Rename to `TX_BUFFER_SIZE` or share as `BUF_SIZE`.
